# GPTBridge Governance Constitution (Level 0 Authority)

This document is the Level 0 Minimal Core. It defines the hierarchy, authority, and immutable principles. Implementation details are prohibited here.

## 1. Governance as Infrastructure
Governance is not documentation; it is runtime infrastructure. It must be versioned, enforceable, and auditable.

## 2. Platform Survivability
The Renderer and MainShell are the highest priority. The platform must remain observable and recoverable even if all underlying services fail.

## 3. Singularity of Truth
Every system aspect (Alias, Startup Flow, Runtime Registry, Governance Index) must have exactly one authoritative source.

## 4. Hierarchy of Power
*   **Level 0: Constitution** (This document) - Hierarchy, Authority, Survivability.
*   **Level 1: Core Governance Domains** - Runtime, Startup, Services, Security, etc.
*   **Level 2: Enforcement Layer** - Checkers, Build Gates, Runtime Validation.
*   **Level 3: Operational Workflow** - Specific coding patterns and dev workflows.
*   **Level 4: Migration / Deprecated** - Legacy sunset and transition logic.

## 5. AI Boundaries
AI agents are prohibited from altering architectural authority, introducing shadow runtimes, or performing unauthorized wide-scale rewrites.
